# 追い越し検出ツール 使い方資料

## 1. このツールでできること

自転車に取り付けた GoPro 動画から、自動車に追い越された候補区間を検出し、候補の確認、LiDAR 点群の選択、速度・離隔距離などの集計用データ出力までを行う。

主な出力は次のとおり。

- 追い越し候補一覧: `out/overtaking_events.csv`
- 目視確認後の候補一覧: `out/reviewed_overtaking_events.csv`
- 候補確認用クリップ: `out/clips/`
- LiDAR 選択点群: `out/lidar/overtaking_XXX.csv`
- 最近接点の軌跡: `out/lidar_edit/overtaking_XXX_edit.csv`
- 解析付き Excel: `out/lidar_edit/overtaking_XXX_edit_analysis.xlsx`
- 最終集計 Excel: `out/final/<出力フォルダ名>_final.xlsx`

## 2. 起動方法

通常は GUI を使う。

```powershell
.\run.bat
```

起動環境だけ確認する場合は次を実行する。

```powershell
.\run.bat --check
```

GitHub から取得した場合もポータブル配布版でも、利用者が実行するファイルは `run.bat` のみ。初回起動時に環境が不足していれば自動で準備する。

## 3. GUI の基本ワークフロー

### Step 1: 動画検出

1. GoPro 動画を選択する。
2. 撮影方向を選択する。
   - 前方撮影: `front`
   - 後方撮影: `rear`
3. 出力フォルダを指定する。
4. 検出を開始する。

検出が終わると `overtaking_events.csv` と、設定に応じて候補クリップが作成される。

撮影方向は検出結果に大きく影響する。日本の左側通行では追い越し車は自転車の右側を抜けるため、前方撮影と後方撮影で画面上の動きが異なる。

- 前方撮影: 車両が画面右端付近から現れ、中央側へ流れる。
- 後方撮影: 後方映像は左右が反転するため、車両が中央付近から左端へ大きくなりながら流れる。

### Step 2: クリップ確認

検出候補を目視で確認し、各候補に状態を付ける。

- 採用: 正しい追い越しとして残す。
- 除外: 誤検知として後続工程から外す。
- 未確認: 判断を保留する。

既定のショートカットは次のとおり。

| キー | 操作 |
|---|---|
| `A` | 採用 |
| `D` | 除外 |
| `W` | 未確認に戻す |
| `B` | 車種を軽に変更 |
| `N` | 車種を普通に変更 |
| `M` | 車種を大型に変更 |
| `Space` | 再生 / 一時停止 |
| `←` / `→` | 前後の候補へ移動 |

採用・除外キーと車種キーは GUI のキー設定から変更できる。設定は `gui_settings.json` に保存される。

確認後は「レビュー結果を CSV 保存」を実行する。除外した候補は `reviewed_overtaking_events.csv` には出力されない。

### Step 3: PCAP・時刻同期

1. Velodyne の PCAP ファイルを読み込む。
2. GoPro GPS が有効な動画では「GPS 自動同期」を実行する。
3. GoPro 動画時刻と LiDAR の経過時刻を合わせるためのオフセットを確認する。

同期後、レビュー済み候補には VeloView 側の時刻が付与される。候補時刻へ移動して LiDAR 点群を確認しやすくなる。

GPS 自動同期が使えない場合は、既知の対応時刻をもとにオフセットを手動入力する。

### Step 4: 点群選択・出力

1. 対象の追い越し候補を選択する。
2. 候補時刻付近の LiDAR フレームへ移動する。
3. 上面図で対象車両の点群を矩形選択する。
4. 必要に応じて対象レーザー番号を指定する。
5. 開始フレームから終了フレームまで、車両を追うように矩形を調整して記録する。
6. `overtaking_XXX.csv` として出力する。

出力先は常に検出出力フォルダ配下の `lidar/` になる。

```text
out/
  lidar/
    overtaking_001.csv
```

LiDAR CSV の 9 列目と 10 列目は、従来工程と同じ `Points:0`、`Points:1` として出力される。

### Step 5: edit CSV・解析 Excel 生成

LiDAR 選択点群から、各フレームの最近接点を抽出した `edit.csv` を生成する。

GUI では対象 CSV を選択して edit CSV 生成を実行する。全候補分をまとめて生成することもできる。

生成物は次の場所に作成される。

```text
out/
  lidar_edit/
    overtaking_001_edit.csv
    overtaking_001_edit_analysis.xlsx
```

`edit_analysis.xlsx` には、フェーズ区分、離隔距離、通過時間、相対速度、TTC などの分析値が含まれる。

### Step 6: 最終 Excel 出力

必要な項目を入力して最終 Excel を生成する。

- 整理 ID
- データ収集日
- 被験者名または ID
- PCAP ファイル
- オフセット
- 危険感
- 車種

出力先は次のとおり。

```text
out/final/<出力フォルダ名>_final.xlsx
```

最終 Excel には、離隔距離、自転車速度、自動車速度、動画時刻、LiDAR 時刻、車種などが出力される。

## 4. コマンドラインでの動画検出

GUI を使わずに動画検出だけ実行する場合は、次のように実行する。

```powershell
# 後方撮影、CSV のみ
.venv\Scripts\python.exe detect_overtaking.py 入力動画.mp4 --view rear

# 前方撮影、候補クリップと可視化動画も出力
.venv\Scripts\python.exe detect_overtaking.py 入力動画.mp4 --view front --clip --overlay

# 精度優先で大きい YOLO モデルを使う
.venv\Scripts\python.exe detect_overtaking.py 入力動画.mp4 --view rear --model yolov8m.pt --conf 0.35
```

`run.bat` も利用できる。

```powershell
.\run.bat input.mp4 rear
.\run.bat --check
```

## 5. よく調整する設定

| 設定 | 既定値 | 用途 |
|---|---:|---|
| `--model` | `yolov8n.pt` | YOLO モデル。大きいモデルほど精度重視、処理は遅くなる。 |
| `--conf` | `0.30` | 車両検出の信頼度しきい値。 |
| `--imgsz` | `640` | 推論解像度。下げると高速化するが精度が落ちやすい。 |
| `--batch` | `32` | GPU 推論のバッチ数。VRAM 不足時は下げる。 |
| `--stop-flow` | `0.8` | 自転車が走行中か停止中かを分ける背景フローしきい値。 |
| `--min-area-ratio` | `1.15` | 接近・離脱に伴う bbox 面積変化の最低倍率。 |
| `--min-vertical-motion` | `0.025` | 道路奥行き方向の最低移動量。 |
| `--clip` | off | 候補クリップを切り出す。 |
| `--overlay` | off | 検出枠付き動画を出力する。 |

取りこぼしが多い場合は、まず `--min-area-ratio` と `--min-vertical-motion` を小さくする。停止中の誤検知が多い場合は、`overtaking_events.csv` の `ego_motion` を確認して `--stop-flow` を調整する。

## 6. 入出力フォルダの目安

標準的な作業後のフォルダ構成は次のようになる。

```text
out/
  overtaking_events.csv
  reviewed_overtaking_events.csv
  clips/
    event_001_right.mp4
  gps/
    <動画名>/
      telemetry_gps.xlsx
  lidar/
    overtaking_001.csv
  lidar_edit/
    overtaking_001_edit.csv
    overtaking_001_edit_analysis.xlsx
  final/
    <出力フォルダ名>_final.xlsx
```

## 7. 注意点

- GoPro GPS が無効な動画では GPS 自動同期は使えない。
- LiDAR は classic PCAP の標準 1206-byte Velodyne データパケットを対象にしている。
- 対応 LiDAR は VLP-16、HDL-32E、VLP-32C。
- 単眼 GoPro の bbox 面積は近さの相対指標であり、実距離ではない。
- 正確な離隔距離や速度の算出には、LiDAR 点群の選択と edit CSV の生成が必要。
- 車種は自動判定後に GUI で手動修正できる。逆光、ぼけ、画面端の歪みがある場合は目視確認を優先する。
